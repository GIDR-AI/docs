# Create Gidr

{/* AUTO:BEGIN */}


## Overview

To check create GIDR flow is working as expected

<Info>
This feature allows you to accomplish the task described above. Follow the procedures below to get started.
</Info>

## Prerequisites

- At least one GIDR should be created for the user to access the "Create GIDR" module.
- User should be in home page with all modules pre-loaded
- User should be in the Create GIDR screen.

## Procedure

<Steps>

<Step title="Verify that the teams page has the GIDR module.">

Verify that the teams page has the GIDR module.

**Expected result:** The GIDR module should be displayed and defaultly selected on the page.

</Step>

<Step title="Verify that the list of GIDR template cards is dis...">

Verify that the list of GIDR template cards is displayed.

**Expected result:** The GIDR template cards should be displayed and clickable.

</Step>

<Step title="Click the 'AI assistant' template card">

Click the "AI assistant" template card

**Expected result:** It should be clickable, and upon clicking, it should open the 'Create GIDR' popup.

</Step>

<Step title="Verify that the  'Create GIDR' popup is displayed">

Verify that the  "Create GIDR" popup is displayed

**Expected result:** The popup should display with the header "Create GIDR", Close icon, "GIDR Name", placeholders for "Enter description", and a "Next" button.

</Step>

<Step title="Enter GIDR name and Description">

Enter GIDR name and Description

**Input:** `Name: SampleDescription: Testing sample`

**Expected result:** The Name and Description fields should be editable

</Step>

<Step title="Click the 'Next' button">

Click the "Next" button

**Expected result:** "Next" button should be enabled and clickable, on click it should display "Connect your data" Popup

</Step>

<Step title="Verify that the 'Connect your data' popup is displ...">

Verify that the "Connect your data" popup is displayed

**Expected result:** The popup should display with the header (Connect your data), a close icon, radio buttons (Files, URLs, Integrations), a click-to-upload button, and "Back" and "Next" buttons. Also that the "Files" radio button is selected by default.

</Step>

<Step title="Click the 'Click to upload' button">

Click the "Click to upload" button

**Expected result:** The "Click to upload" container should be clickable. Upon clicking, it should open the File explorer window.

</Step>

<Step title="Select the File from the File explorer">

Select the File from the File explorer

**Expected result:** Choose a file to upload, and ensure the uploaded file is displayed.

</Step>

<Step title="Click the 'Next' button">

Click the "Next" button

**Expected result:** The "Next" button should be clickable. Upon clicking, it should redirect to GIDR page.

</Step>

</Steps>

## Parameters

The following parameters are used in this process:

- `Name: SampleDescription: Testing sample`
- `Name:  TestDescription: GIDR testing`
