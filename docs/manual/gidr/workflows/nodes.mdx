## Procedure

<Steps>

<Step title="Login as a super Admin">

Login as a super Admin

**Input:** `super@gidr.aiPassword1!`

**Expected result:** Upon successful login, the <strong data-start="138" data-end="177" data-is-only-node="" id="isPasted">Super Admin Dashboard (Home Screen)</strong> should be displayed.

</Step>

<Step title="Click the <strong data-start='67' data-end='83' id...">

Click the <strong data-start="67" data-end="83" id="isPasted">“Three Dots”</strong> (options) button for the newly created organization.

**Input:** `Organization Name: new`

**Expected result:** It should displaying available options like Edit, Disable, Impersonate, Delete.

</Step>

<Step title="Click the <strong data-start='88' data-end='105' i...">

Click the <strong data-start="88" data-end="105" id="isPasted">“Impersonate”</strong> option.

**Expected result:** Upon clicking It should navigate to user screen and display the organization’s dashboard/home screen.

</Step>

<Step title="Click the <strong data-start='76' data-end='92' id...">

Click the <strong data-start="76" data-end="92" id="isPasted">User Profile</strong> icon and navigate to the <strong data-start="118" data-end="143">Organization Settings</strong> screen.

**Expected result:** The Organization Settings screen should be displayed, showing configuration options, and the Studio toggle button should be turned on.

</Step>

<Step title="Click the <strong data-start='99' data-end='117' i...">

Click the <strong data-start="99" data-end="117" id="isPasted">“Sandbox Team”</strong> option and create a new <strong data-start="142" data-end="150" data-is-only-node="">GIDR</strong>.

**Expected result:** A new <strong data-start="182" data-end="190" id="isPasted">GIDR</strong> should be successfully created under the Sandbox Team, and a confirmation/success message should be displayed.

</Step>

<Step title="Open the any&nbsp;<strong data-start='72' data-end...">

</Step>

<Step title="Verify the GIDRs list and Verify that the <strong ...">

Verify the GIDRs list and Verify that the <strong id="isPasted">Flow Gadget</strong> is present.

**Expected result:** The Flow Gadget should be displayed in the GIDRs list with the correct name and should be visible to the user.

</Step>

<Step title="Click the <strong data-start='65' data-end='75' id...">

Click the <strong data-start="65" data-end="75" id="isPasted">Studio</strong> tab inside the GIDR.

**Expected result:** The <strong data-start="125" data-end="142" id="isPasted">Studio screen</strong> should open, and the <strong data-start="164" data-end="182">“Add Workflow”</strong> button should be visible and enabled for use.

</Step>

<Step title="Click the <strong data-start='97' data-end='115' i...">

Click the <strong data-start="97" data-end="115" id="isPasted">Add Workflow (+)</strong> button.

**Expected result:** Upon clicking It should display the <strong data-start="183" data-end="209" id="isPasted">Create Workflow popup and</strong> allowing the user to create a new workflow.

</Step>

<Step title="Enter a valid workflow name and Click the <strong ...">

Enter a valid workflow name and Click the <strong data-start="135" data-end="143" id="isPasted">Save</strong> button.

**Input:** `Name: Workflow 4`

**Expected result:** The workflow should be <strong id="isPasted">saved successfully</strong>, and a confirmation message should be displayed. The newly created workflow should also appear in the workflow list and also display the all available nodes.

</Step>

</Steps>
